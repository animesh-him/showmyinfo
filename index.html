Exposed information What it means How to protect yourself
User‑Agent / Platform Browser name, version, OS – lets a site serve exploits for that exact combination. Keep browsers/OS up‑to‑date. Use privacy‑focused browsers (Firefox, Brave) or extensions that spoof/shorten the UA string.
Language(s) Preferred UI language(s). Helps infer location or ethnicity when combined with IP. Set a generic language (e.g., “en‑US”) or use an extension that randomises language headers.
Cookies enabled / Do‑Not‑Track Shows whether third‑party cookies work; DNT flag signals you don’t want tracking. Block third‑party cookies in browser settings; enable Do‑Not‑Track and add a tracker‑blocking extension.
Screen size, colour depth, pixel depth, device‑pixel‑ratio Resolution & display characteristics – classic “screen‑fingerprint”. Use a privacy‑oriented browser or extension that reports a common resolution (e.g., 1024 × 768) and rounds DPR.
Hardware concurrency (CPU cores) Number of logical CPU cores – narrows device class (mobile vs. desktop). Some extensions let you spoof a lower core count (e.g., report 2 cores).
Device memory (RAM) * Approximate RAM in GB – further refines fingerprint. Hide the value via privacy extensions or use a browser that omits it.
Timezone offset Local UTC offset – can be combined with IP to locate you. Force UTC (offset = 0) with a privacy extension or use a VPN that also adjusts your system clock.
Locale‑aware date/time strings Formatted strings reveal regional settings (e.g., month names). Set a generic locale or use an extension that rewrites `Intl` objects.
Public IP address Your outward‑facing IP – the strongest identifier, reveals ISP & rough location. Browse through a reputable VPN, Tor, or proxy. Enable IPv6 privacy extensions on your OS.
WebGL fingerprint (GPU vendor/renderer) GPU model & driver details – often unique per device. Block or spoof WebGL via extensions (CanvasBlocker, Trace) or disable WebGL in browser flags.
Battery status * Charging state & level – can act as a slowly‑changing fingerprint. Most browsers hide this on insecure origins; you can also disable the Battery API in settings or via extensions.
Touch‑point count Indicates presence of a touchscreen – helps differentiate mobile vs. desktop. Usually low‑risk; if desired, spoof the value with a privacy extension.
Media devices (cameras, microphones, speakers) List of available input/output devices (labels are empty until permission is granted). Reveals hardware capability. Block `navigator.mediaDevices.enumerateDevices` via a privacy extension or simply deny permission prompts.

* Only exposed on secure (HTTPS) pages and when the browser permits the API.
